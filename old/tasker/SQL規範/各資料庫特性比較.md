# NoSQL特性
NoSQL定義和**有無執行個體**並無關聯性，其主要專注在結構本身是否有關聯，以下為其特性
- 無join、tran，操作為其獨特API接口 
- 結構不需事先定義(類似弱型別概念，可以擁有同一列有不同欄情況)
- 結構可自由修改(類似弱型別概念)
- 因為結構限制寬鬆，可用需求來異動結構存儲方式
- 分佈式架構
- 應用情境:
- 存取log作分析:因資料量龐大**存取問題**，且**格式改動**後因資料量龐大不可能跟著修正
